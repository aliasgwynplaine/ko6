9  \brief Management of double chained lists
10 
11  This file contains the functions (inline & define) necessary to manage double chained lists.
12  These are based on embedded structures (list_t) with two pointers each which are linked together.
13  - A list consists of only two types of objects:
14  1. a type for the root structure with an embedded list_t member
15  2. a type for all items in the list also with an embedded list_t member
16  - The list_t member of the root is connected to the list_t member that are in the items
17  - it is possible to calculate the address of the container, by knowing:
18  - the memory address of the embedded structure (list_t),
19  - the type C of the item that contains the (list_t) member and
20  - the member name of the embedded member (list_t) in the item
21 
22  struct root_s { struct item_s { Note that the list_t member can be placed
23  type1 member_1; type1 member_a; anywhere in the root_s and item_s structures
24  ... ... and it can be nammed anything. The chosen names
25  list_t root; list_t item; depends onthe use of the structures.
26  typen member_n; typem members_m; Generally, root_s and item_s do not have the
27  }; }; same size (warning, the scheme can be misleading)
